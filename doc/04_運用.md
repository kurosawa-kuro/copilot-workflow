# 運用ガイド

> ドキュメント更新手順・PR 運用・整合性チェック。
> 手順変更時に更新する。本リポジトリはドキュメント専用のため、ビルド / デプロイ / インフラ操作は存在しない。

---

## 1. 前提（開始前の確認）

- **編集ツール**: テキストエディタ + git
- **権威順位を確認してから編集する**（`doc/README.md §2`）
- **正本は `src/01_workflow-guide.md`**。ステップ・ゲート定義はここを起点にする

---

## 2. 定常更新パターン

### 2.1 ワークフローステップの変更

1. `src/01_workflow-guide.md` を先に更新（正本）
2. `src/02_prompt-templates.md` を **同一変更で** 更新（ステップに 1:1 対応するプロンプトを合わせる）
3. mermaid 図・チェックリスト・プロンプト番号が一致しているか確認
4. `doc/01_仕様と設計.md` の §2（8 ステップ定義表）が古くなる場合は更新

### 2.2 新タスク種別の追加

例: 「Cloud Run Job」に加えて新種別を追加する場合。

1. `src/02_prompt-templates.md` の **前提欄**（ファイル先頭）に新種別を追記
2. **Step 4 プロンプト**のタスク種別リストを更新
3. **4-ゴールテンプレート**に新種別の成功定義テーブルを追加（成功条件 / 確認方法の表形式）
4. `src/01_workflow-guide.md` の Step 4 サブ箇条（成功定義の確認方法）を更新
5. `doc/01_仕様と設計.md` の共通前提・対象タスク種別を更新

### 2.3 ファイル追加・リネーム

1. `doc/03_実装カタログ.md` のディレクトリ構成とファイル一覧を更新
2. `README.md` のドキュメント構成表を更新
3. `CLAUDE.md` のファイルツリー（```ブロック内）を更新
4. `doc/README.md §1` のファイル一覧を更新

### 2.4 AI エージェント向けルール変更

- `CLAUDE.md` / `AGENTS.md` を更新
- 両ファイルの前提・不変条件が一致しているか確認

- [ ] 連動すべきファイルを全て同一 PR で修正した（`doc/README.md §3` 参照）

- [ ] mermaid 図のノードと番号付きチェックリストのステップが一致している

## 4. 整合性チェック（drift 発見・修正手順）

### 症状: workflow-guide と prompt-templates のステップ番号がズレている

```bash
# 番号確認（番号付きリストの先頭行を抽出）
grep -n "^- \[ \] \*\*[0-9]" src/01_workflow-guide.md
grep -n "^### [0-9]" src/02_prompt-templates.md
```

対応: `src/01_workflow-guide.md` を正として `src/02_prompt-templates.md` を合わせる。

### 症状: 実在しないファイルが 03_実装カタログ.md に記載されている

```bash
ls -R .
```

対応: 実際のファイル構成を確認し、カタログを実態に合わせる。

### 症状: 旧プロジェクトの内容（資格戦略等）が残っている

```bash
grep -r "CAPM\|PSPO\|PMLE\|資格" doc/ src/
```

対応: 該当箇所を copilot-workflow の内容に書き直す。
