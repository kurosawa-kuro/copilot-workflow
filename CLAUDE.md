# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## このリポジトリの性質

コードを持たない **ドキュメント専用** リポジトリ。GitHub Copilot を使った開発作業を、メンバーが START（仕様書）から GOAL（PR）まで進めるための運用フローとプロンプト雛形をまとめている。ビルド / lint / test / 実行コマンドは存在しない。成果物は Markdown のみ。

```
README.md   … リポジトリ入口（目的 / ワークフロー全体像 / src 導線）
src/
  01_workflow-guide.md    … 運用ガイド本体（フロー図 + チェックリスト）
  02_copilot-prompts.md  … 各ステップで Copilot に渡すプロンプト雛形（GCP / MLOps / Terraform 向け）
  03_spec-template.md        … 依頼仕様書テンプレート（リーダー/クライアントが記入する入力側）
  04_plan-templates.md      … 作業計画書テンプレート（ゴール・成功定義・種別別雛形）
  05_sample-memo-template.md … 実装サンプルメモテンプレート（メンバー自身用・提出不要）
```

git は初期化済みだがまだ 1 コミットも無い（全ファイル untracked、ブランチは `master`）。

## ドキュメントが規定するワークフロー

3 つのファイルは密結合しており、関連する変更は同一変更でまとめて更新する。

- **`01_workflow-guide.md`** が正本。**確認フェーズ（実装しない）→ ゲート① → 実装フェーズ → ゲート②** の 2 ゲート制を mermaid フロー図とチェックリスト（8 ステップ）で定義する。
- **`02_copilot-prompts.md`** は `01_workflow-guide.md` の 8 ステップそれぞれに 1:1 対応するプロンプト例集。ステップの追加・削除・番号変更は両ファイルに反映する。
- **`04_plan-templates.md`** は作業計画書の記入雛形。ゴール・成功定義・種別別確認方法を収録。タスク種別追加時は `02_copilot-prompts.md` の前提欄とともに更新する。

このワークフローを編集・運用する際に外してはならない不変条件:

1. **確認フェーズではプロダクションコードもテストコードも書かない。** 提出物は作業計画書のみ（作業計画書は Claude Code / Codex / Cursor の **plan mode** に相当する工程・成果物）。
2. **テストファースト。** 実装フェーズはステップ5（テスト作成）→ステップ7（実装）の順で、実装はテストを満たす最小限。
3. **「実装サンプルメモ」は任意・自分用でリーダー提出不要。** 作業計画書（必須提出物）と混同しない。
4. **2 つのゲートはリーダー承認待ち。** 差し戻し先はステップ7が原則だが、原因に応じて前工程（5/4/1）へ戻る。
5. プロンプト集は **GCP（Vertex AI / BigQuery / Cloud Run Jobs / Elastic Cloud）/ Terraform を主軸とした MLOps 基盤** が前提（冪等性 / 再実行安全性 / GCP API クォータ / コスト管理 / パイプライン失敗リカバリ / ログ追跡を重視）。モデル性能改善・scikit-learn 等の DS 作業はスコープ外。
6. **幻覚対策＝grounding。** AIが挙げた関数 / API / ライブラリ / 設定キー / DBカラムは実コマンド（grep・import・公式doc・実行）の**出力**で裏取りする。実行者は人間でもAIエージェントでもよいが、禁止は「実行せず記憶で実在を断定すること」。出力を残せないものは `[要検証]`。ステップ2に実在確認、ステップ8レビュー最上位に「実在性」を置く。詳細手順は 02_copilot-prompts.md「実在確認」に集約（guide 側はクイックに保つ）。
