# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## このリポジトリの性質

コードを持たない **ドキュメント専用** リポジトリ。GitHub Copilot を使った開発作業を、メンバーが START（仕様書）から GOAL（PR）まで進めるための運用フローとプロンプト雛形をまとめている。ビルド / lint / test / 実行コマンドは存在しない。成果物は Markdown のみ。

```
README.md   … リポジトリ入口（目的 / ワークフロー全体像 / docs 導線）
docs/
  workflow-guide.md     … 運用ガイド本体（フロー図 + チェックリスト）
  prompt-templates.md   … 各ステップで Copilot に渡すプロンプト雛形（Python バッチ向け）
```

git は初期化済みだがまだ 1 コミットも無い（全ファイル untracked、ブランチは `master`）。

## ドキュメントが規定するワークフロー

2 つのファイルは密結合しており、片方を編集したらもう片方との整合を必ず確認する。

- **`workflow-guide.md`** が正本。**確認フェーズ（実装しない）→ ゲート① → 実装フェーズ → ゲート②** の 2 ゲート制を mermaid フロー図とチェックリスト（8 ステップ）で定義する。
- **`prompt-templates.md`** は `workflow-guide.md` の 8 ステップそれぞれに 1:1 対応するプロンプト例集。ステップの追加・削除・番号変更は両ファイルに反映する。

このワークフローを編集・運用する際に外してはならない不変条件:

1. **確認フェーズではプロダクションコードもテストコードも書かない。** 提出物は作業計画書のみ。
2. **テストファースト。** 実装フェーズはステップ5（テスト作成）→ステップ7（実装）の順で、実装はテストを満たす最小限。
3. **「実装サンプルメモ」は任意・自分用でリーダー提出不要。** 作業計画書（必須提出物）と混同しない。
4. **2 つのゲートはリーダー承認待ち。** 差し戻し先はステップ7が原則だが、原因に応じて前工程（5/4/1）へ戻る。
5. プロンプト集は **Python の非バックエンド・バッチ処理** が前提（冪等性 / 再実行安全性 / チャンク処理 / 異常系 / ログ追跡を重視）。
6. **幻覚対策＝grounding。** AIが挙げた関数 / API / ライブラリ / 設定キー / DBカラムは実物（grep・import・公式doc・実行）で裏取りする。AIに聞き返して確認しない。ステップ2に実在確認、ステップ8レビュー最上位に「実在性」を置く。
